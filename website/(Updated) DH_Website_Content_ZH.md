# 二、比賽資訊 / Competition Info

## 2.1 關於比賽

### 2.1.1 比賽簡介

「2026 理大人工智能 X 數位人文獎」是香港理工大學人文學院專為本科生舉辦的旗艦型校內活動，聚焦於人工智能與古典研究的跨界融合，旨在鼓勵學生深入研習中國古典文獻、經典思想、歷史文化、翻譯及語言分析。

### 2.1.2 主題說明：AI & Classics

本屆主題為「AI & Classics」。所謂 Classics，在本比賽中可廣義理解為古典文本、經典思想、古代歷史文化、古籍圖像、經典翻譯與相關知識傳播。參賽作品可圍繞中國古典文獻與文化，也可延伸至古典語言、古典接受史、經典詮釋或跨文化翻譯等課題，但須清楚說明其與「經典」或「古典學習」的關聯。

### 2.1.3 作品方向

參賽作品可參考以下方向，以下示例僅供參考，並不構成限制。

| 方向 | 說明 |
|---|---|
| **生成式AI與古典文本** | 應用生成式AI協助經典文本導讀、翻譯比較、註釋設計或互動問答 |
| **自然語言處理與資料視覺化** | 運用NLP、NER、知識圖譜、GIS及資料視覺化分析古典文本或歷史材料 |
| **AI Agent工作流程** | 設計自動化AI agent工作流程，串聯史料提取、資料整理、內容分析至數位呈現 |
| **AI輔助數位人文工具** | 結合CBDB、MARKUS、DocuSky等工具進行文本標註、人物地名抽取及視覺化分析 |
| **AI教學工具** | 設計以古典內容為核心的AI教學工具、AI tutor或數位學習平台 |
| **資料集與模型** | 建立與古典文獻、人物、地名、圖像相關的資料集、模型或檢索工具 |
| **AR/VR與多模態** | 利用AR/VR、空間運算或多模態敘事重構古典學習與文化傳播體驗 |

## 2.2 參賽資格及組隊

### 2.2.1 參賽資格

參賽者必須為香港理工大學人文學院的全日制本科生，並就讀於以下課程之一：

| 課程 |
|---|
| 人文學科課程組合本科入學計劃 |
| 中國歷史及文化（榮譽）文學士 |
| 英文及應用語言學（榮譽）文學士 |
| 言語治療（榮譽）理學士 |

主辦單位保留核實學生身份及組隊資格之權利。

### 2.2.2 組隊規則

| 規則 | 說明 |
|---|---|
| **隊伍人數** | 可個人參賽，亦可組成2至4人隊伍，每隊最多4人 |
| **跨學科組隊** | 鼓勵跨學科組隊；若成員來自不同課程，須於報名時說明各成員的負責範疇 |
| **成員限制** | 每名學生只可作為一隊之正式成員參賽 |
| **提交限制** | 每隊只可提交一份作品 |

## 2.3 提交要求

### 2.3.1 參賽組別

本比賽設三個參賽組別。參賽者須於報名時選擇所屬組別。主辦單位將按作品實際內容及提交時所選組別進行評審。若作品橫跨兩組別，參賽者須在作品簡介中說明其主要評審定位。

| 組別 | 定位 | 適合作品示例 |
|---|---|---|
| **A組** | 善用GenAI工具 | 以現成AI工具設計經典文本導讀、翻譯比較、課堂活動、互動問答、教材框架或學習資源 |
| **B組** | 客製化AI工具、AI Agent或AI Agent Workflow | 建立AI tutor、文本分析工具、資料抽取工具、知識圖譜、資料集、檢索系統或AI Agent Workflow原型 |
| **C組** | 多模態、沉浸式AI與經典 | 運用AR/VR、地圖、圖像辨識、多模態生成、數位展覽或空間敘事重構經典學習與文化傳播體驗 |

### 2.3.2 可接受的作品形式

所有作品須同時具備兩項基本要求：其一，能清楚回應「AI & Classics」主題；其二，能說明作品的教育價值、研究價值或公共傳播價值。

| 作品類型 | 子方向／說明 |
|---|---|
| **研究型作品** | • **生成式AI與古典文本**：AI協助古典文本分析、史料整理、人物／地名／概念抽取、翻譯比較研究或教學研究 |
| | • **AI輔助數位人文工具**：結合CBDB、MARKUS、DocuSky等工具進行文本標註、人物地名抽取及視覺化分析 |
| | • **自然語言處理與資料視覺化**：運用NLP、NER、知識圖譜、GIS及資料視覺化分析古典文本或歷史材料 |
| **工具型作品** | 網站原型、AI Agent、AI chatbot、AI tutor、資料視覺化儀表板、檢索系統、資料集或模型 |
| **教學型作品** | 課程單元設計、互動教材、學習任務、AI輔助經典導讀方案或評估工具 |
| **展示型作品** | 短片、數位展覽、故事地圖、AR/VR體驗、多模態敘事作品、圖像分類或圖文互動原型 |
| **AI Agent Workflow型作品** | 以AI agent或多agent協作流程為核心，展示從史料／文本提取、資料清理與標註、知識抽取、分析推理，到視覺化、故事地圖、數位展覽或教學輸出的完整工作流程。作品可包括workflow架構圖、提示設計、工具串接、資料流設計、示範平台或prototype |

## 2.5 評審

### 2.5.1 評審準則

本比賽從五個維度進行評審，結合國際AI教育比賽常見評審方向與數位人文評估框架。如作品屬B組或C組且具較強技術原型元素，評審可酌情更重視可行性與功能性；如屬研究型或教學型，則會更重視學術準確性與教育設計。

| 評審項目 | 比重 | 說明 |
|---|---|---|
| **問題界定與Classics／Humanities關聯** | 20% | 是否清楚指出經典文本、古典文化或古代歷史／語言問題，並說明教育或研究上的實際需求 |
| **可行性與功能性／技術實踐** | 20% | 作品是否可操作性、技術設計是否合理；如屬工具原型，是否能有效展示其功能 |
| **創新與創意** | 20% | AI與人文內容的結合方式是否新穎，有否超越一般示範式使用 |
| **學術質素與研究嚴謹度** | 25% | 文本、史料、譯本、概念與文獻運用是否準確，論證是否有方法、有依據 |
| **社會影響、可持續性與負責任AI** | 15% | 是否考慮倫理、版權、偏差、可持續性、可擴展性及教育公平問題 |

### 2.5.2 評審程序

評審分三個階段進行：

| 階段 | 說明 |
|---|---|
| **第一階段：初審** | 所有初賽作品由評審根據公布準則，就書面及媒體材料進行初審 |
| **第二階段：入圍及修訂** | 公布入圍名單，入圍隊伍按評審意見修訂作品 |
| **第三階段：決賽簡報及示範** | 入圍隊伍須向跨學科評審團進行簡報及示範（如適用） |

### 2.5.3 評審團組成

每份入圍作品原則上由不少於3名評審審閱。評審背景將盡量涵蓋歷史、語言、文學、翻譯、語言科學、數位人文或AI應用等範疇。主辦單位可邀請校外專家參與，以加強公平性及跨學科視角。

## 2.7 規則與常見問題

### 2.7.1 AI使用與學術誠信

所有作品必須為參賽者原創。以下規定適用於所有提交作品：

| 規定 | 說明 |
|---|---|
| **原創性** | 作品不得直接抄襲、冒用或重複提交曾於其他比賽獲獎之作品 |
| **AI使用申報** | 如作品使用了生成式AI，必須申報所使用的工具、提示方式、生成內容範圍及人手修訂情況 |
| **未申報AI內容** | 如主辦單位發現未申報之AI生成內容、虛假來源、捏造文獻或嚴重學術不當行為，可取消參賽或得獎資格 |
| **版權與倫理** | 參賽者須遵守版權、私隱、資料保護及研究倫理要求；如使用第三方文本、圖像、地圖、資料集或模型，須確保合法授權或合理引用 |
| **歷史與文化材料** | 涉及歷史人物、地理資料、古籍圖像或文化遺產材料者，須注意準確性、脈絡及潛在偏差，並鼓勵在作品中反思AI的限制 |

### 2.7.2 知識產權與公開展示

| 項目 | 說明 |
|---|---|
| **著作權歸屬** | 作品之著作權及知識產權原則上屬參賽者所有 |
| **公開展示同意** | 參賽者同意主辦單位在註明作者／隊伍的前提下，於網站、社交媒體、教學活動、展覽、宣傳刊物及相關教育用途展示其作品 |
| **主辦單位權利** | 主辦單位保留對比賽細則、時程及評審安排作出修訂、延長或解釋之權利 |

### 2.7.3 常見問題

**問1：誰可以參賽？**
香港理工大學人文學院全日制本科生，就讀於四個合資格課程之一。詳見「參賽資格及組隊規則」頁面。

**問2：可以個人參賽嗎？**
可以。參賽者可個人參賽，亦可組成2至4人隊伍。每名學生只可作為一隊之正式成員參賽。

**問3：我需要有編程經驗嗎？**
毋須具備高階編程能力。參賽者可從A組入手，善用現成GenAI工具。比賽期間亦會提供培訓工作坊，支援不同程度的參賽者。

**問4：我的作品可以橫跨兩個組別嗎？**
可以，但參賽者須在作品摘要中說明其主要評審定位。

**問5：AI使用聲明須包含什麼內容？**
聲明須列明所使用的生成式AI工具、其用途、生成內容範圍，以及人手修改、驗證與判斷的程度。

**問6：我的知識產權會保留嗎？**
會。作品之著作權及知識產權原則上屬參賽者所有。主辦單位可在註明作者的前提下公開展示作品。
